02
The Problem
Manual outreach doesn't scale
Our GTM team was spending hours on prospect research and competitive analysis that followed repeatable patterns — the kind of work agents should be doing.
Manual prospect sourcing
Finding and qualifying leads from LinkedIn required hours of manual search, copy-paste, and data cleanup per campaign.
No competitive visibility
Understanding what competitors were doing in paid media meant manually browsing ad libraries with no structured analysis.
Disconnected tools
Scraping, enrichment, analysis, and outreach each lived in separate tools with no shared pipeline.
No agent access
Automation was possible in theory but impossible in practice — no CLI, no skill manifests, no structured output for agents to consume.

06
Key Design Decisions
What shaped the architecture
01
CLI as equal citizen
Most platforms add an API later. Here the CLI and web UI both consume the same service layer — feature parity is architectural, not aspirational.
02
ServiceResult over exceptions
Every function returns { data, error }. Makes error paths explicit, eliminates try-catch chains, works identically across web and CLI.
03
Skill manifests for discovery
JSON manifests describe every command's parameters, output schema, and examples. Agents can plan before executing.
04
Org isolation at service layer
Every query includes org_id. No RLS dependency, no cross-tenant data leakage, auditable from a single point.
